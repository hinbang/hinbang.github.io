  2. KCP
    1. 牺牲带宽换取网络传输速率 为什么这么说呢？
    2. KCP没有规定下层传输协议，一般用UDP作为下层传输协议
    3. KCP提高速率的方法
      • RTO/Retransmission Timeout ： TCP 2xRTO，UDP 1.5xRTO 。
      • 快重传机制 ：发送端发送了1,2,3,4,5几个包，然后收到远端的ACK: 1, 3, 4, 5，当收到ACK3时， KCP知道2被跳过1次，收到ACK4时，知道2被跳过了2次，此时可以认为2号丢失，不用等超时，直接重传2号包，大大改善了丢包时的传输速度。
      • 选择重传 ：TCP丢包时会全部重传从丢的那个包开始以后的数据，KCP是选择性重传，只重传真正丢失的数据包。
      • 非延迟ACK ：TCP为了充分利用带宽，延迟发送ACK（NODELAY都没用），这样超时计算会算出较大的RTT时间，延长了丢包时的判断过程。KCP的ACK是否延迟发送可以调节。

  1. FileZilla没有解压缩功能.即使你使用的FileZilla,也需要与SSH命令行搭配使用.使用SSH命令行连接远端服务器和解压缩上传文件命令：
    ssh 用户名@IP -p 端口号

    如果你要解压缩文件的话,cd到相关目录之后
    参考Mac terminal下解压缩命令大全

  2. 什么是nas，什么是群晖？
    Nas是（Network Attached Storage网络附属存储）的简称
    群晖作为nas，通过DiskStation Manager (DSM)是网页界面的直觉式操作系统，可帮助用户管理存于Synology DiskStation和RackStation里的文件、照片、音乐、视频以及其他重要内容。DiskStation Manager可实现的不仅是存储资料，用户还可以自行设置应用程序与选择所需要的服务。

  1. firewalld 防火墙（CentOS 7）
    service firewalld restart 重启
    service firewalld start 开启
    service firewalld stop 关闭
    systemctl status firewall 服务状态
    firewall-cmd –state 状态
    firewall-cmd –list-all 查看防火墙状态
    firewall-cmd –zone=public –add-port=80/tcp –permanent （–permanent永久生效，没有此参数重启后失效）
